KONF_CONFIG_DIR=./config ./target/release/konf-backend &
curl http://localhost:8000/v1/health

Config, not code

A product is a directory of YAML and markdown that defines a complete agent: workflows, tools, memory, prompts. No application code required.

Local-first & self-hostable

Runs as a single Rust binary against an embedded redb file. No Postgres, no Docker, no cloud. Bring your own LLM keys.

Capability-based runtime

Process table, capability enforcement, and namespace injection. Tools run inside an ExecutionScope — least-privilege by default.

Pluggable tools

Compiled Rust, MCP, HTTP, and (soon) WASM tool adapters all implement one Tool trait. Add a tool, don't fork the engine.

MCP & HTTP shells

Expose products to Claude Desktop, Cursor, and other MCP clients via konf-mcp, or to apps via the konf-backend REST + SSE API.

Open by default

BSL-1.1 today, Apache-2.0 on 2030-04-01. Free for personal, development, and internal production use.
